# 消融实验报告

这份报告只讨论“策略选择”问题：  
同一任务下，`PE / RAG / FT` 各自带来什么，组合后又意味着什么。

## 1. 实验口径

### 1.1 统一任务

- 任务：Celery 跨文件依赖分析
- 正式评测集：`data/eval_cases.json`
- 样本数：`54`
- 评分口径：FQN 级别精确匹配

### 1.2 说明

当前仓库的消融矩阵分为两层：

1. **已经完成的正式结果**
2. **尚未补齐的 Qwen ablation 空位**

也就是说，这份报告不会伪造缺失数据，而是明确告诉你“已经能下什么结论、还差什么才能闭环”。

## 3. 当前最重要的三条消融结论

### 3.1 PE 是最强的独立策略

在 GPT-5.4 上：

- `Baseline 0.2815`
- `PE only 0.6062`

这说明：

- 对强模型而言，Prompt 口径和输出约束就是第一生产力
- 如果预算有限，最先做的不是微调，也不是检索，而是把 PE 做对

### 3.2 FT 单独有效，但远不如 PE + FT

在 Qwen 上：

- `Baseline 0.0370`
- `FT only 0.0932`
- `PE + FT 0.4315`

这说明：

- 微调单独确实提升了领域适配能力
- 但如果没有 PE，模型依然难以稳定把内部知识转成正确 FQN 输出
- 真正有生产价值的是 `PE + FT`，而不是单独 FT

### 3.3 RAG 的价值不是全局提分，而是针对 hard 场景

在 GPT-5.4 上：

- `Overall +0.0157`
- `Hard +0.1392`
- `Easy -0.1241`

这说明：

- RAG 更像“条件触发模块”
- 不是所有 case 都值得吃额外检索和上下文成本

## 4. 现阶段可以确认的策略边界

### 4.1 商业模型

当前最优路线非常明确：

- **GPT-5.4 + PE**

原因：

- 它已经给出全仓库最强正式分数 `0.6062`
- 成本远低于“全量 RAG + 更长上下文”
- 结论稳定、没有旧 embedding 版本依赖

### 4.2 开源模型

如果只看当前严格、稳妥、可复现的开源结果：

- **Qwen PE + FT = 0.4315**

如果允许把旧版 RAG+FT 当作参考上限：

- **Qwen PE + RAG + FT = 0.4435**

但这里必须强调：

- 当前 `0.4435` 结果早于最新 Google embedding
- 在严格最终版交付里，仍建议重跑后再下最终结论

### 4.3 策略适用边界

| 场景 | 推荐策略 | 原因 |
|------|------|------|
| easy / 简单 alias / 普通 re-export | PE | 收益最大、成本最低 |
| medium / 命名空间混淆 / 动态字符串 | PE + RAG 或 PE + FT | 需要更多结构或领域模式 |
| hard / bootstep / finalize / fixup | PE + RAG + FT | 需要检索和内化知识同时存在 |

## 5. 当前还缺的矩阵空位

### 5.1 缺失项

当前还没有正式新版结果的 Qwen 组合是：

- `Qwen PE only`
- `Qwen RAG only`
- `Qwen PE + RAG`
- 建议重跑 `Qwen PE + RAG + FT`

### 5.2 为什么这几项必须补

因为现在还能明确回答：

- PE 在 GPT 上很强
- FT 在 Qwen 上有帮助
- RAG 在 GPT hard case 上有帮助

但还不能完全闭环回答：

- `Qwen` 上 PE 单独能拉到多高
- `Qwen` 上 RAG 单独到底值不值得
- `Qwen` 的最佳组合是 `PE + FT` 还是 `PE + RAG + FT` 的最新版

## 6. 当前最稳的项目级结论

### 6.1 如果导师现在就看结果

最稳的说法是：

1. 商业模型上界：`GPT-5.4 + PE = 0.6062`
2. 当前正式开源最稳路线：`Qwen PE + FT = 0.4315`
3. RAG 的主要价值：修 `hard / Type A / Type E`

### 6.2 如果补齐剩余 Qwen 实验后

这份消融矩阵就能真正回答：

- `PE only / RAG only / FT only / PE+RAG / PE+FT / All`

也就是题目要求的完整策略矩阵。

## 7. 最终建议

### 7.1 现在就能落地的结论

- **默认商业模型方案**：`GPT-5.4 + PE`
- **默认开源模型方案**：`Qwen PE + FT`

### 7.2 后续补跑优先级

建议顺序：

1. `Qwen PE only`
2. `Qwen RAG only`
3. `Qwen PE + RAG`
4. `Qwen PE + RAG + FT`（最新版）

补跑说明见：

- [`../docs/qwen_remaining_runs_20260328.md`](../docs/qwen_remaining_runs_20260328.md)

## 8. 结论

这轮消融已经足够明确三件事：

1. **PE 是最大贡献者**
2. **FT 的价值需要通过 PE 才能稳定释放**
3. **RAG 的价值是“选择性修复复杂场景”而不是“平均分万能药”**

剩余工作只集中在 Qwen 的少数空位，不再影响项目主线结论的成立。
