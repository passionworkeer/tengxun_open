### 项目名称：【RepoMind】多语言代码仓库分析能力评测体系构建

==================== 内容介绍 ====================

本项目旨在系统性构建 RepoMind MCP 工具的多语言评测体系。主要事项包括：

1. 语言覆盖拓展：扩展至 Go、Python、Java、C/C++、Rust、TypeScript 等主流语言，每种语言覆盖不少于 10 个典型测试场景（调用链分析、历史 commit 追踪、跨文件引用搜索、全局变量追踪、函数级 diff 分析等）。
2. 测试集标准化构建：每个场景构建不低于 50 条高质量用例，涵盖正向验证、边界条件、异常处理等维度，用例需贴合真实开源项目特征。
3. 多语言特性适配：针对各语言独特特性（Go interface 组合、Java 继承体系、Python 动态类型、C++ 模板元编程、Rust ownership 等）设计专项测试，确保分析能力跨语言有效。
4. 评测指标设计：建立统一评测维度——调用链完整性（召回率/准确率）、历史追踪准确性、跨文件引用覆盖率、语义搜索相关性评分等，形成可量化评测报告。
5. 自动化评测流程：设计可复用的测试框架，支持批量运行、自动采集结果、对比计算指标。

==================== 最终输出/期望项目成果 ====================

1. 多语言测试集：覆盖 ≥6 种语言，每种 ≥10 个场景，每场景 ≥50 条用例，总计 ≥3000 条。
2. 测试场景设计文档：各语言的场景定义、用例设计思路、覆盖的 MCP 工具能力说明。
3. 评测报告：按语言和场景维度输出准确率、召回率、响应质量等量化指标，识别能力短板与优势。
4. 自动化评测脚本：一键批量调用 MCP 接口、结果采集与指标计算，输出结构化报告。
5. 语言特性分析报告：总结各语言在仓库分析场景下的特性差异，为后续优化提供方向建议。

==================== 对学生能力和知识背景的要求 ====================

1. 熟悉至少 3 种主流编程语言（Go、Python、Java、C/C++、TypeScript 任意组合），能阅读理解对应语言的开源项目代码。
2. 了解 MCP 协议基本概念，理解工具调用方式和返回结构。
3. 具备软件测试基础（等价类划分、边界值分析等），能独立设计高质量测试场景和数据。
4. 熟悉 Git 核心概念（commit、diff、blame、log），能基于真实仓库构造测试用例。
5. 具备 Python/Shell 脚本能力，能编写自动化测试与结果采集脚本。有代码静态分析、调用链分析或 AST 经验者优先。

==================== 实践目标 ====================

1. 掌握 RepoMind MCP 工具的能力边界，理解其核心分析链路（历史追踪、调用链构建、跨文件搜索、语义检索等）。
2. 建立多语言代码分析对比视角，理解静态/动态类型、编译/解释型等差异对分析精度的影响。
3. 培养系统化的测试集设计能力，形成从场景定义到自动化执行的完整评测方法论。
4. 通过真实开源项目实践，积累多语言项目结构、依赖管理、构建体系的实战经验。
5. 输出可复用的评测框架和测试资产，为 RepoMind 多语言能力迭代提供质量保障基础。

### 项目名称：【RepoMind】基于微调/PE/RAG 的领域分析效果优化

==================== 内容介绍 ====================

本项目以已有评测基线分数为起点，通过微调（Fine-tuning）、提示工程（PE）和检索增强生成（RAG）三大路径，系统性提升 RepoMind 在目标领域方向上的分析效果。主要事项包括：

1. 基线分析与瓶颈定位：分析目标领域（调用链分析、历史追踪、跨文件搜索、语义检索等方向之一）的得分分布，识别低分场景共性特征，明确优化切入点。
2. 模型微调：构建领域专项微调数据集（≥500 条），采用 LoRA/QLoRA 等高效微调方法进行领域适配训练，设计训练/验证/测试集划分策略，监控过拟合。
3. 提示工程（PE）：优化 System Prompt（角色定义、输出格式约束）、Few-shot 示例设计、Chain-of-Thought 推理引导、输出后处理规则（过滤、排序、去重）。
4. 检索增强生成（RAG）：优化代码片段向量化索引（Embedding 选型调优）、检索策略（语义+关键词混合）、上下文窗口管理、检索结果与生成模型的融合策略。
5. 消融实验：分别评估单一手段和组合策略的效果增益，以基线分数为对照组，输出量化分数提升报告。

==================== 最终输出/期望项目成果 ====================

1. 优化后的模型：微调权重文件（LoRA adapter 或完整模型），附 Model Card（训练数据规模、参数、基线对比分数）。
2. PE 方案文档：优化前后 Prompt 对比、Few-shot 示例库（≥20 条）、CoT 模板、后处理规则，及各优化点的效果数据。
3. RAG 方案：完整 Pipeline 配置与代码（向量索引、检索策略、上下文管理），附检索质量评估报告。
4. 量化评测报告：微调/PE/RAG 独立与组合效果对比，含分数提升幅度、场景得分明细、消融实验结论。
5. 可复现实验包：完整实验代码、配置、训练脚本、评测脚本及 README。

==================== 对学生能力和知识背景的要求 ====================

1. 具备 LLM 基础知识（Transformer、注意力机制、Token 化），了解主流开源模型（LLaMA、Qwen、DeepSeek 等）。
2. 熟悉至少一种微调方法（LoRA/QLoRA/Full Fine-tuning），掌握 HuggingFace Transformers、PEFT 框架。
3. 掌握 PE 技巧（System Prompt、Few-shot、Chain-of-Thought），能系统性设计和评估 Prompt 方案。
4. 了解 RAG 架构与流程，熟悉向量数据库（FAISS/Milvus/Chroma）和 Embedding 模型。
5. 具备 Python + PyTorch 编程能力，能编写数据处理、训练、评测脚本。
6. 了解 MCP 协议和 RepoMind 工具调用方式。有代码仓库分析或代码智能经验者优先。

==================== 实践目标 ====================

1. 建立"发现问题 → 定位瓶颈 → 设计方案 → 实验验证 → 效果评估"的闭环优化思维。
2. 掌握微调/PE/RAG 三大优化手段的适用场景、实施方法和效果边界。
3. 积累代码仓库分析垂直领域的模型优化经验，理解代码语义理解等场景的独特挑战。
4. 培养严格的实验设计能力（对照组、消融实验、统计显著性），形成数据驱动决策习惯。
5. 输出可复现实验资产，为 RepoMind 领域能力提升提供可落地的优化方案。

### 项目名称：【RepoMind】基于业务场景的理想态测试用例设计与竞品验证

==================== 内容介绍 ====================

本项目围绕 RepoMind MCP 工具的真实业务场景，基于"理想态"反向设计高质量测试用例，并在竞品上交叉验证，确保测试集的客观性和区分度。主要事项包括：

1. 业务场景与理想态定义：梳理核心场景——配置参数分析（参数名 → 取值范围、函数调用链、所属场景、修改验证）、函数变更追踪（函数名 → 历史修改、关联 Issue、变更原因）、代码依赖分析（模块/文件 → 上下游依赖、调用链路、影响面）、安全漏洞定位（CVE → 受影响代码、修复建议、关联 commit）等，定义各场景理想态输出标准。
2. 配置参数分析深度设计（重点场景）：输入配置参数名称，理想态输出包含：(a) 参数语义理解（含义、类型、取值范围）；(b) 函数调用追踪（读取/修改该参数的函数及链路）；(c) 所属场景识别（业务模块归属）；(d) 修改验证（影响范围、关联功能、风险提示）。
3. 用例标准化设计：每条用例包含场景分类、输入、理想态输出、评判标准、难度等级，覆盖正向/边界/反向维度，总计 ≥500 条。
4. 用例去重与质控：建立相似度检测机制（TF-IDF/Embedding 相似度），确保高度相近用例不超过 3 条。
5. 竞品交叉验证：在 RepoMind 及 ≥2 款竞品工具上执行测试集，分数波动控制在 5% 以内，验证客观性。

==================== 最终输出/期望项目成果 ====================

1. 理想态定义文档：覆盖 ≥5 个核心场景，每个含场景描述、典型输入、理想态输出标准、评判维度及权重。
2. 测试用例集：≥500 条，按场景分类，每条含完整输入/理想态输出/评判标准，相似度已检测。
3. 竞品验证报告：各工具场景维度与总分对比，波动分析（目标 ≤5%）。
4. 用例质量报告：覆盖度分析、相似度检测结果、理想态覆盖完整性自查。
5. 自动化评测与去重工具：批量执行、结果采集、得分计算、相似度检测脚本。

==================== 对学生能力和知识背景的要求 ====================

1. 具备软件测试理论基础（等价类划分、边界值分析、场景法等），能独立设计高质量测试用例。
2. 了解代码仓库分析核心概念（配置参数管理、函数调用链、代码依赖），能阅读 ≥2 种主流语言的项目代码。
3. 了解 MCP 协议，理解 RepoMind 工具调用方式和返回结构。
4. 具备文本相似度计算基础（TF-IDF、Embedding 相似度），能进行用例去重检测。
5. 具备 Python 脚本能力，能开发自动化评测、结果采集、相似度检测工具。
6. 了解主流代码分析工具（Sourcegraph、GitHub Copilot、Cursor 等），能设计公平对比实验。

==================== 实践目标 ====================

1. 掌握"理想态驱动"的测试设计方法论，从业务场景定义理想效果再反向推导用例。
2. 深入理解代码仓库分析核心场景，建立对 MCP 工具业务价值的系统认知。
3. 培养测试集质量控制能力（去重、覆盖度、相似度检测）。
4. 通过竞品交叉验证，培养公平 Benchmark 设计能力。
5. 输出可复用的测试资产和自动化工具，为 RepoMind 持续评测提供标准化基准。
